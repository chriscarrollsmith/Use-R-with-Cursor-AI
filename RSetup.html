Installing R and an IDE

  1. Install R for your operating system from CRAN.

  2. (Optional but recommended) Install an R-friendly Integrated Development Environment (IDE):

  • RStudio - the classic choice with lots of user-friendly R tooling
  • Cursor with the R extension and the Quarto extension - full-featured AI-first IDE based on VSCode
  • Positron - Experimental RStudio successor forked from VSCode with robust R and Quarto support
  1. Open an R terminal (optionally in your IDE).

Setting up the R library path

  1. Create a file called .Renviron in your home directory by running:
file.create(file.path(Sys.getenv("HOME"), ".Renviron"))
  1. Open the .Renviron file in your default text editor by running:
file.edit(file.path(Sys.getenv("HOME"), ".Renviron"))
  1. Add the following to your .Renviron file, replacing {version} with the version of R you have installed:
R_LIBS_USER=~/Documents/R/library/{version}

This is the path to the library where R will install user-level packages for this version of R. Make sure to save the .Renviron file when you are finished editing it.

  1. Load the .Renviron file by running:
readRenviron("~/.Renviron")
  1. Create the library directory if it doesnâ€™t exist by running:
dir.create(Sys.getenv("R_LIBS_USER"), recursive = TRUE)
  1. Run the following command to verify that the library path was set correctly:
.libPaths()

Setting up the R profile

  1. To ensure that .Renviron is sourced on R startup, create a file called .Rprofile in your home directory by running:
file.create(file.path(Sys.getenv("HOME"), ".Rprofile"))
  1. Edit the .Rprofile file in your default text editor by running:
file.edit(file.path(Sys.getenv("HOME"), ".Rprofile"))
  1. To load the .Renviron file on startup, add the following to your .Rprofile:
if (file.exists("~/.Renviron")) {
  readRenviron("~/.Renviron")
}

This ensures that the .Renviron file is read when R is started.

  1. To prevent R from always prompting you to choose a mirror when you install packages, also add the following to your .Rprofile (replace the URL with that of your favorite CRAN mirror):
options(repos = c(CRAN = "https://lib.stat.cmu.edu/R/CRAN/"))

  1. Save the .Rprofile file and restart R by closing and reopening the terminal.

  2. Run the following command to verify that the user profile was set up correctly:

.libPaths()
options()$repos

Installing user-level packages

  1. Install user-level packages. I recommend the following:
install.packages("tidyverse")
install.packages("devtools")
install.packages("renv")

A Note on project-level packages

  1. I recommend using the renv package to manage project-level packages rather than installing them directly into the user library. This allows you to isolate project-specific dependencies to avoid conflicts with the user-level setup and to ensure reproducible builds across different systems.

    1. To initialize renv in a project, run the following command in the root of your project:
    renv::init()

    This will create a lockfile called renv.lock, an renv/library directory, and a project-specific .Rprofile that will activate renv when you open R from within the project folder.

    1. You can now install project-level packages not just from CRAN, but also from GitHub and Bioconductor, using renv::install().

    2. With your libraries installed, run renv::snapshot() to update the renv.lock file, and make sure to commit the renv.lock file to git.

    3. Anyone cloning your repository will be able to install the exact same versions of the packages you used by running renv::restore(). This prevents random package updates from breaking your builds.

    4. Update the renv.lock file periodically by running renv::update(). If it breaks your build, you can always roll back the update with renv::revert().

    5. For perfect reproducibility, combine with Docker.
